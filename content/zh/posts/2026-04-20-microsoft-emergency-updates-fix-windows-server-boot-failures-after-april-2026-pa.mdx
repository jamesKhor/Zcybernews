## 执行摘要
微软发布了紧急非计划内的（OOB）更新，以解决安装了2026年4月补丁星期二安全更新的Windows Server 2019和2022系统的关键启动和认证失败问题。根据微软的说法，这个问题特别影响配置为使用Kerberos密钥分发中心（KDC）代理功能的域控制器和服务器，导致它们无法重新启动或认证用户。公司建议受影响的管理员立即应用OOB更新，这些更新可通过Windows Update、Microsoft Update Catalog和Windows Server Update Services（WSUS）获得。

## 技术分析
有问题的更新，被识别为2026年4月8日的安全发布（Windows Server 2019的KB5043155和Windows Server 2022的KB5043157），在Kerberos认证子系统中引入了一个缺陷。微软的咨询声明，当启用KDC代理功能时，漏洞就会显现。安装了4月更新后，域控制器可能会遇到启动失败或在重启过程中变得无响应。对于能够启动的系统，认证请求可能会失败，有效地锁定用户访问网络资源。根本原因是安全更新本身内部的代码缺陷，而不是正在被利用的新漏洞。2026年4月17日发布的紧急OOB更新包含了受影响二进制文件的修正版本，以恢复系统的正常运行。

## 威胁行为者背景
此事件源于合法微软安全更新中的软件缺陷。在可用的来源中没有证据将此与恶意活动或威胁行为者的利用联系起来。影响是应用官方补丁的直接结果。

## 缓解措施与建议
受影响的Windows Server 2019和2022系统的管理员必须应用相应的OOB更新：Windows Server 2019的KB5044439或Windows Server 2022的KB5044441。微软表示这些是累积更新，取代了有问题的4月补丁。这些更新可以通过标准分发渠道安装。对于无法启动的系统，微软建议启动到安全模式或使用恢复媒体从4月更新之前拍摄的备份中恢复系统，或启动修复安装。公司没有提供不涉及应用新更新或执行恢复操作的替代方案或缓解措施。
