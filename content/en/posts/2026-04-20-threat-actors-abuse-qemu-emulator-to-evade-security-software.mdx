## Executive Summary
Threat actors are actively exploiting the legitimate QEMU machine emulator to bypass security controls and deploy malware. Security researchers have identified at least two separate campaigns where attackers used QEMU to load and execute ransomware and remote access trojan (RAT) payloads directly in memory, a technique that leverages the emulator's trusted status to evade endpoint detection and response (EDR) tools.

## Technical Analysis
According to a report from SecurityWeek, attackers are abusing the open-source QEMU emulator as a living-off-the-land binary (LOLBin) for defense evasion. The core technique involves using QEMU's `-device` argument to load a malicious firmware image, which is then executed within the QEMU process space. This allows the attackers to run their payloads under the guise of a legitimate, trusted virtualization tool, significantly reducing the likelihood of detection by security software that may whitelist or not deeply inspect such processes.

The report details two distinct campaigns. In one, attackers used QEMU to deploy a ransomware payload. In the other, a remote access tool (RAT) was delivered using the same method. By executing the malicious code from within QEMU, the attackers avoid writing the final payload to disk, enabling a fileless execution strategy that complicates forensic analysis and traditional signature-based detection.

## Indicators of Compromise
None identified in source material.

## Tactics, Techniques & Procedures
The primary technique observed is the abuse of QEMU for defense evasion (T1218.011: Signed Binary Proxy Execution). Attackers are using the emulator as a proxy to load and execute malicious firmware or code. This falls under the broader Living off the Land (LOTL) strategy, where attackers leverage pre-installed, legitimate software to carry out malicious activities. The specific command-line invocation of QEMU with a `-device` argument pointing to a malicious file is a key behavioral indicator.

## Threat Actor Context
The source material confirms at least two separate campaigns are using this technique but does not attribute them to known threat groups or provide information on their origins or motivations beyond the deployment of ransomware and RATs. The use of a sophisticated, non-standard LOLBin like QEMU suggests actors with a moderate to high level of technical capability who are actively researching evasion methods.

## Mitigations & Recommendations
Security teams should monitor for unusual QEMU process execution, particularly command lines containing the `-device` argument pointing to unfamiliar firmware files (e.g., `*.bin`, `*.rom`). Application control or endpoint detection and response (EDR) policies should be configured to alert on or block QEMU execution from non-standard paths or by non-privileged users, especially in environments where the tool is not part of an approved workflow. As QEMU is a legitimate systems tool, outright blocking may not be feasible; therefore, behavioral detection focusing on the sequence of QEMU launching and subsequently making anomalous network connections or performing file encryption is critical.
