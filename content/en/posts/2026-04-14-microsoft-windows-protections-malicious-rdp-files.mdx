## Executive Summary
Microsoft has implemented new security features in Windows to mitigate a persistent phishing technique that abuses Remote Desktop Protocol (RDP) connection files. The update, detailed in a recent Microsoft security blog post, introduces explicit user warnings and disables risky resource-sharing options by default when opening `.rdp` files from untrusted sources. This change directly targets campaigns where threat actors distribute malicious RDP files to harvest credentials or establish initial access to victim systems.

## Technical Analysis
The core of the threat involves weaponizing the legitimate `.rdp` file format, which stores configuration data for establishing Remote Desktop connections. According to Microsoft, attackers craft these files to include directives that automatically connect to a malicious remote server or, more commonly, enable the sharing of local client resources—like drives and printers—with the remote host. When a user opens a malicious `.rdp` file, Windows would previously establish the connection with these resource-sharing settings enabled, potentially allowing an attacker to deploy malware directly to the user's shared drives or access sensitive files.

The new protections, which began rolling out with a recent Windows update, alter this behavior in two key ways. First, when an `.rdp` file is opened from an untrusted source—such as an email attachment or a download from the internet—Windows now displays a clear security warning dialog. This warning explicitly states the file's origin and the risks of connecting to an unknown remote PC. Second, and more significantly, the system now defaults to disabling the sharing of local client drives, printers, and clipboard with the remote computer for connections initiated from these untrusted files. Users must manually re-enable these features through the warning dialog if required, creating a conscious security decision point.

## Indicators of Compromise
None identified at this time.

## Tactics, Techniques & Procedures
The primary technique observed is **Phishing (T1566)**, specifically using a malicious `.rdp` file as the payload. The associated procedures involve:
*   **Weaponized Document Delivery:** Distributing `.rdp` files via email or other social engineering lures, often disguised as legitimate business documents or invoices.
*   **Resource Sharing Abuse:** Configuring the `.rdp` file with `drivestoredirect:s:*` and `redirectprinters:i:1` parameters to enable sharing of the victim's local drives and printers with the attacker-controlled RDP server.
*   **Credential Harvesting:** Relying on the Windows Credential Prompt that appears during the RDP connection attempt to capture the user's login credentials for the remote system or, through reuse, their corporate network credentials.
*   **Initial Access:** Using the established RDP session and shared resource pathways to gain a foothold on the victim's network for further exploitation.

## Threat Actor Context
Microsoft has not attributed this specific attack vector to a named threat actor or group in its public advisory. The technique is considered a tool-agnostic, commodity method likely employed by a broad range of cybercriminals, from low-skill phishing operators to more advanced ransomware affiliates, due to its simplicity and effectiveness. The abuse of trusted, native Windows functionality like RDP files makes these campaigns particularly evasive to traditional detection methods that focus on executable malware.

## Mitigations & Recommendations
The primary mitigation is applying the latest Windows updates from Microsoft, which enable the new protective behavior by default. Organizations should ensure these updates are deployed across their environments. Additional defensive measures include:
*   **User Training:** Educate employees on the risks of opening unexpected `.rdp` files or any file type from untrusted sources, even if they appear to be non-executable documents.
*   **Application Control:** Use Windows Defender Application Control or similar policies to restrict the execution of `mstsc.exe` (the Remote Desktop client) to authorized users and systems where it is strictly necessary for business functions.
*   **Network Segmentation:** Restrict outbound RDP traffic (TCP port 3389) from standard user workstations to the internet, limiting the ability to connect to external malicious RDP servers.
*   **Enhanced Monitoring:** Audit Windows Event Logs for Event ID 1149, which is generated when the new `.rdp` file security warning is displayed, to identify potential attack attempts.
