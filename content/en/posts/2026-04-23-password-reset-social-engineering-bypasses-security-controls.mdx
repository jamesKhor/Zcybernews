## Executive Summary

Password reset workflows — a standard IT security practice — are increasingly exploited by attackers as a vector for account takeover, according to research published by Specops Software. The company's analysis shows that social engineering of helpdesk personnel can turn a seemingly legitimate reset request into full compromise, bypassing multi-factor authentication (MFA) and other controls. The technique does not rely on technical vulnerabilities but on procedural gaps in identity verification during the reset process.

## Technical Analysis

Specops Software's research, shared with BleepingComputer, details how attackers initiate a password reset by contacting the helpdesk posing as the legitimate user. The attacker provides readily available personal information — such as the user's name, department, or manager — to satisfy low-rigor verification checks. Once the helpdesk resets the password, the attacker gains immediate access to the account, including any MFA-protected resources, because the reset itself is treated as a trusted action.

The attack chain works because many organizations rely on knowledge-based authentication (KBA) questions whose answers are often discoverable via OSINT or prior breaches. Specops noted that even organizations using MFA are vulnerable if the reset process itself is not MFA-protected. The technique is particularly effective against helpdesk teams that lack caller-ID verification or callback confirmation procedures.

## Mitigations & Recommendations

Defenders should implement out-of-band verification for all password reset requests — such as calling the user back on a pre-registered phone number or sending a push notification to a registered device. Organizations should also move away from knowledge-based authentication and adopt hardware-backed tokens or biometric verification for helpdesk interactions. Monitoring for clusters of password reset requests from the same IP or targeting the same department can help detect coordinated social engineering campaigns.
