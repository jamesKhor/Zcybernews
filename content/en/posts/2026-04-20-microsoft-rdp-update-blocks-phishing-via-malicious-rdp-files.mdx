## Executive Summary
Microsoft has deployed a security-focused behavioral change to the Windows Remote Desktop Connection client (MSTSC) in its April 2026 updates, introducing new warning dialogs to counter phishing campaigns that weaponize malicious Remote Desktop Protocol (.rdp) files. The update, part of the April 2026 Patch Tuesday cycle, directly addresses a technique where attackers trick users into opening crafted .rdp files that connect to attacker-controlled servers, harvesting credentials.

## Technical Analysis
The update modifies the `mstsc.exe` client to display a new security warning when a user opens an .rdp file that initiates a connection to a remote host. According to the source, the dialog warns users of an "unknown connection" and requires explicit user consent to proceed. This change targets a specific attack vector: threat actors have been embedding malicious connection parameters within .rdp files, which are then distributed via phishing emails or malicious websites. When an unsuspecting user opens the file, the Remote Desktop client automatically attempts to connect to the attacker's server, presenting a fake login prompt designed to harvest Windows credentials. The new warning interrupts this automated flow, forcing user interaction and providing an opportunity to abort the connection.

## Indicators of Compromise
None identified in source material.

## Tactics, Techniques & Procedures
The primary technique mitigated by this update is the use of malicious .rdp files for credential harvesting (T1589.001: Gather Victim Identity Information). Attackers craft these files to point to infrastructure they control, often using social engineering (T1566: Phishing) to lure victims into opening them. The attack relies on the default behavior of the Windows shell associating .rdp files with `mstsc.exe`, which would previously initiate the connection with minimal user prompting.

## Threat Actor Context
The source material does not attribute this .rdp file phishing technique to a specific named threat actor or group. It describes the method as an increasing trend among threat actors broadly, indicating its adoption as a low-cost, effective credential access vector.

## Mitigations & Recommendations
The primary mitigation is to apply the latest Windows security updates from April 2026, which contain the updated `mstsc.exe` client. Organizations should ensure these patches are deployed across all endpoints. Administrators can also consider implementing Group Policy to restrict the execution of .rdp files from untrusted locations or mandating Network Level Authentication (NLA) for all RDP connections, which provides an additional authentication step before a session is established. User awareness training should highlight the risks of opening unsolicited .rdp file attachments.
